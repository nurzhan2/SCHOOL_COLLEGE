РАЗВЁРТЫВАНИЕ (для разработчика)
================================
Сайт теперь работает на Node-сервере (server.js) — он раздаёт
страницы И обслуживает раздел «События» с админкой.

ЗАПУСК ЛОКАЛЬНО:
  npm install
  npm start        (откроется на http://localhost:3000)

RAILWAY:
  railway.json уже настроен: startCommand = "node server.js".
  При деплое Railway сам сделает npm install (express, multer).
  Пароль админки можно задать переменной окружения ADMIN_KEY
  (по умолчанию "shkol2026").

РАЗДЕЛ «СОБЫТИЯ»:
  - На сайте: блок «События» на главной берёт данные из /api/events.
    Если событий нет или сервер статический — показывается запасной
    блок со ссылкой на ВКонтакте.
  - В админке (/admin → вкладка «События»): добавление (фото + дата +
    заголовок + текст + ссылка «Подробнее») и удаление.
  - Хранение: data/events.json, фото — assets/events/.

ВАЖНО (персистентность на Railway):
  Файлы, записанные во время работы, сбрасываются при каждом
  редеплое. Чтобы события и загруженные фото сохранялись:
  подключите Railway Volume и примонтируйте его, задав переменную
  окружения DATA_DIR на путь тома (например /data), а папку
  assets/events лучше тоже вынести на том. Без тома события живут
  до следующего деплоя.
